    机器学习——基础篇1

    发表于 2026-01-25 分类于 AI 阅读次数：

    本篇全部基于周志华《机器学习》（西瓜书）第1-2章内容。

    前言——机器学习基本思维框架

    归纳 vs 演绎

    • 归纳： 从特殊$\Longrightarrow$一般 （泛化过程）
      • 广义：从样例中学习
      • 狭义（概念学习）：从训练数据中学得某个概念
    • 演绎：从一般$\Longrightarrow$特殊 （特化过程）

    机器学习发展简史

    1. 推理期（符号主义、逻辑推理）
    2. 知识期（专家系统、知识工程）
    3. 从样例学习期（决策树、规则学习、ILP归纳逻辑程序设计、集成学习）
    4. 连接主义复兴（神经网络、BP算法）
    5. 大数据 + 深度学习热潮（2012年后主导）

    总体趋势：逻辑推理 $\Rightarrow$ 知识表示 $\Rightarrow$ 数据驱动 $\Rightarrow$ 表示学习（深度）
    每一次范式转变背后都有三大驱动力：计算能力跃升 + 数据规模爆炸 + 理论/算法突破。

    模型评估与选择

    过拟合与欠拟合

    学习器把训练样本的个别特性当作所有潜在样本的普遍性质 → 泛化能力下降 = 过拟合
    相反，学习能力不足，连训练集都拟合不好 = 欠拟合

    误差的三个层次名称

    • 训练误差 / 经验误差：训练集上的误差
    • 泛化误差：真实新样本上的期望误差（我们最关心，但无法直接获得）
    • 测试误差 ≈ 对泛化误差的无偏估计（理想情况下）

      常用实验评估方法（数据集划分策略）

    方法 核心做法 优点 缺点 / 适用场景 备注
    留出法 D 随机划分为 S（训练）+ T（测试） 简单、直接 单次划分随机性大；样本少时测试集太小 常重复多次随机划分取平均
    k折交叉验证 D 均分为 k 份，轮流做验证集，其余训练 评估更稳定，利用数据更充分 计算量约为留出法的 k 倍 k=5 或 10 最常用；k=n → 留一法
    自助法 有放回采样 m 次，得到新训练集（约63.2%原始数据），剩余≈36.8%做测试 可从较小数据集产生多个不同训练集；包外估计 引入采样偏差（有放回） 特别适合集成学习、数据集很小时

    参数 vs 超参数

    • 模型参数（需要学习）：权重、偏置等（深度模型可达亿级）
    • 超参数（人为设定）：学习率、树深度、正则系数、k折的k等（通常10个以内）

    调参目标：找到一组超参数，使模型在验证集上表现最好。

    常见性能度量

    回归任务

    • 最常用：均方误差 MSE（或 RMSE）

    分类任务

    • 错误率 = 分类错误样本数 / 总样本数
    • 精度（准确率） = 1 – 错误率

    不平衡场景下更重要的指标

    • 查准率 P (Precision) = TP / (TP + FP)
    • 查全率 R (Recall) = TP / (TP + FN) 也叫召回率
    • P-R 曲线：横轴 R，纵轴 P
      • 若曲线 A 完全包住曲线 B → A 优于 B
      • 曲线交叉时，可比较曲线下的面积（但不如 F1 直观）
    • 平衡点 BEP（Break-Even Point）：P = R 时的取值（越高越好），常用于简单比较
    • F1 分数 = 2 × (P × R) / (P + R) （调和平均，更平衡）

    ROC 曲线 & AUC

    • 横轴：FPR = FP / (FP + TN)
    • 纵轴：TPR = Recall
    • AUC：ROC 曲线下的面积（0.5=随机，1=完美）
    • ROC 对类别不平衡更鲁棒，P-R 曲线对正例稀疏场景更敏感

    代价敏感学习

    不同类型错误代价不同 → 引入非均等代价矩阵，得到代价敏感错误率或代价曲线

    统计假设检验（模型显著性比较）

    常用方法（西瓜书重点）：

    • 交叉验证 t 检验
    • McNemar 检验（适合配对设计）
    • Friedman 检验 + Nemenyi 事后检验（多算法多数据集比较）

    偏差-方差分解（核心理论视角）——理解泛化误差来源

    对于平方损失（回归最常见），泛化误差可分解为：

    泛化误差 = 偏差² + 方差 + 噪声

    • 偏差（Bias）：刻画学习算法本身的拟合能力（期望预测与真实值差距）
    • 方差（Variance）：刻画同样规模训练集变化引起的模型输出波动（数据扰动敏感度）
    • 噪声（Noise）：学习任务本身难度决定的下界，任何算法都无法突破

    偏差-方差窘境（Bias-Variance Dilemma）：

    • 模型太简单 → 高偏差、低方差（欠拟合）
    • 模型太复杂 → 低偏差、高方差（过拟合）
    • 目标：找到偏差与方差的甜点，最小化两者之和
